\section{Phase 1: Problem Definition}

\subsection{Problem Statement}

Pakistan's used car market suffers from significant price opacity and information asymmetry, creating challenges for all stakeholders. The market is characterized by:

\begin{itemize}[leftmargin=*]
    \item \textbf{Price Inconsistency}: Similar vehicles exhibit substantial price variations (up to 30\%) across different sellers and cities without clear justification.
    \item \textbf{Information Asymmetry}: Sellers possess complete vehicle history while buyers rely on incomplete information, leading to adverse selection and market inefficiency.
    \item \textbf{Market Fragmentation}: Pakistan's used car market spans multiple cities with varying demand-supply dynamics, yet lacks a unified pricing mechanism.
    \item \textbf{Limited Transparency}: Traditional valuation methods depend heavily on subjective dealer assessments rather than data-driven analysis.
\end{itemize}

These challenges result in:
\begin{itemize}
    \item Extended negotiation periods (average 2--3 weeks)
    \item Risk of overpayment or undervaluation
    \item Reduced market liquidity
    \item Increased transaction costs
\end{itemize}

\subsection{Target Audience}

Our solution addresses the needs of three primary stakeholder groups:

\subsubsection{Individual Buyers (70\% of users)}
\textbf{Needs:} Fair market price estimation, confidence in purchase decisions, understanding of price-influencing factors.\\
\textbf{Pain Points:} Fear of overpaying, difficulty assessing vehicle value, limited bargaining power.

\subsubsection{Car Dealers \& Showrooms (20\% of users)}
\textbf{Needs:} Competitive pricing strategies, inventory valuation, market trend analysis.\\
\textbf{Pain Points:} Manual price research, regional price variations, rapid depreciation assessment.

\subsubsection{Financial Institutions (10\% of users)}
\textbf{Needs:} Accurate vehicle valuation for loans, risk assessment, collateral evaluation.\\
\textbf{Pain Points:} Loan default risks, manual appraisal processes, outdated valuation methods.

\subsection{Why AI is Essential}

\subsubsection{Non-Linear Price Relationships}

Used car pricing involves complex, non-linear interactions between features that traditional methods cannot capture:

\begin{equation}
\text{Price} = f(\text{age}, \text{mileage}, \text{brand}, \text{city}, \ldots, \text{interactions})
\end{equation}

where $f$ is a highly non-linear function that machine learning models can approximate effectively.

\subsubsection{Limitations of Rule-Based Systems}

Traditional rule-based approaches fail because:
\begin{itemize}
    \item \textbf{Static Rules}: Cannot adapt to dynamic market conditions.
    \item \textbf{Linear Assumptions}: Depreciation is non-linear (luxury cars depreciate differently than economy cars).
    \item \textbf{Feature Interactions}: Cannot model complex interactions (e.g., high mileage on a luxury brand vs.\ economy brand).
    \item \textbf{Regional Variations}: Manual rules cannot account for city-specific demand-supply dynamics.
\end{itemize}

\subsubsection{Machine Learning Advantages}

AI-powered prediction offers:
\begin{enumerate}
    \item \textbf{Pattern Recognition}: Automatically discovers non-obvious pricing patterns from historical data.
    \item \textbf{Feature Interactions}: Models complex relationships (e.g., $\text{age} \times \text{mileage}$, $\text{brand\_tier} \times \text{engine\_cc}$).
    \item \textbf{Adaptive Learning}: Continuously improves as new market data becomes available.
    \item \textbf{Multi-Factor Analysis}: Simultaneously considers 23+ features (base + engineered).
    \item \textbf{Explainability (SHAP)}: Provides transparent reasoning for predictions, building user trust.
\end{enumerate}

\subsubsection{Why XAI (Explainable AI) is Critical}

For Pakistan's market, model transparency is non-negotiable:
\begin{itemize}
    \item \textbf{Trust Building}: Users need to understand \textit{why} a price is predicted.
    \item \textbf{Regulatory Compliance}: Financial institutions require auditable valuations.
    \item \textbf{User Education}: Helps buyers/sellers understand value drivers.
    \item \textbf{Model Debugging}: Identifies potential biases or errors.
\end{itemize}

SHAP (SHapley Additive exPlanations) provides per-prediction feature contribution analysis, enabling statements like: \textit{``This car is priced Rs~500,000 higher because: imported assembly (+Rs~300,000), low mileage (+Rs~150,000), automatic transmission (+Rs~50,000).''}

\section{Phase 2: Data Collection \& Preprocessing}

\subsection{Data Collection Methodology}

\subsubsection{Data Source}

We collected data from \textbf{PakWheels.com}, Pakistan's largest online automotive marketplace, chosen for:
\begin{itemize}
    \item \textbf{Market Coverage}: 80\%+ of online used car listings in Pakistan.
    \item \textbf{Data Richness}: Comprehensive vehicle specifications and seller information.
    \item \textbf{Geographic Diversity}: Listings from all major Pakistani cities.
    \item \textbf{Active Market}: Real-time, actively traded vehicles.
\end{itemize}

\subsubsection{Web Scraping Approach}

We implemented a custom Python-based scraper using:
\begin{itemize}
    \item \textbf{Libraries}: \texttt{requests}, \texttt{BeautifulSoup4}
    \item \textbf{Rate Limiting}: 2--4 second delays between requests to respect server load.
    \item \textbf{Two-Phase Strategy}:
    \begin{enumerate}
        \item Phase 1: Collect listing URLs from search result pages.
        \item Phase 2: Extract detailed specifications from individual listing pages.
    \end{enumerate}
\end{itemize}

\begin{lstlisting}[language=Python, caption=Price Parsing Function]
def parse_price(text):
    text = text.lower().replace(',', '').strip()
    n = float(re.search(r'(\d+\.?\d*)', text).group(1))
    if 'crore' in text:
        return int(n * 10_000_000)
    if 'lac' in text:
        return int(n * 100_000)
    return int(n)
\end{lstlisting}

\subsection{Data Cleaning Pipeline}

Our data cleaning process consisted of 7 systematic steps:

\subsubsection{Step 1: Duplicate Removal}

\begin{lstlisting}[language=Python, caption=Duplicate Removal]
df.drop_duplicates(inplace=True)
# Removed: 248 duplicates
\end{lstlisting}

\subsubsection{Step 2: Missing Value Handling}

\textbf{Strategy:}
\begin{itemize}
    \item \textbf{Critical columns} (\texttt{make}, \texttt{model}, \texttt{price}): Drop rows.
    \item \textbf{Numeric features}: Group-wise median imputation.
    \item \textbf{Categorical features}: Group-wise mode imputation.
\end{itemize}

\begin{lstlisting}[language=Python, caption=Missing Value Imputation]
df['year'] = df.groupby('make')['year'].transform(
    lambda x: x.fillna(x.median())
)
df['mileage_km'] = df.groupby('year')['mileage_km'].transform(
    lambda x: x.fillna(x.median())
)
df['assembly'] = df.groupby('make')['assembly'].transform(
    lambda x: x.fillna(x.mode()[0] if not x.mode().empty else 'Local')
)
\end{lstlisting}

\subsubsection{Step 3: Data Type Standardization}

\begin{lstlisting}[language=Python, caption=Type Conversion]
df['year']       = df['year'].astype(int)

df['engine_cc']  = df['engine_cc'].astype(int)

df['make']  = df['make'].str.strip().str.title()
df['model'] = df['model'].str.strip().str.title()

\subsubsection{Step 5: Text Cleaning}

\begin{lstlisting}[language=Python, caption=Model Name Cleaning]
def clean_model_name(model):
    unwanted = [
        r'\s+for\s+sale.*$',
        r'\s+in\s+[A-Z].*$',
        r'\s+urgent.*$'
    ]
    for pattern in unwanted:
        model = re.sub(pattern, '', model, flags=re.IGNORECASE)
    return model.strip()

df['model'] = df['model'].apply(clean_model_name)
\end{lstlisting}

\subsection{Feature Engineering Strategy}

We created 12 additional features to capture domain-specific patterns:

\subsubsection{Time-Based Features (3)}

\begin{lstlisting}[language=Python, caption=Time Features]
current_year = 2026

df['car_age'] = current_year - df['year']

df['mileage_per_year'] = df['mileage_km'] / df['car_age'].replace(0, 1)

def age_group(age):
    if age <= 3:   return 'New'
    elif age <= 7: return 'Mid'
    else:          return 'Old'

df['age_group'] = df['car_age'].apply(age_group)
\end{lstlisting}

\textbf{Rationale:} Car depreciation is age-dependent, not year-dependent. A 2020 car in 2026 is 6 years old regardless of the year feature.

\subsubsection{Binary Features (4)}

\begin{lstlisting}[language=Python, caption=Binary Features]
df['is_imported']    = (df['assembly'] == 'Imported').astype(int)
df['is_automatic']   = (df['transmission'] == 'Automatic').astype(int)
df['cross_city_sale']= (df['reg_city'] != df['city']).astype(int)
df['is_hybrid']      = (df['fuel_type'] == 'Hybrid').astype(int)
\end{lstlisting}

\textbf{Rationale:} Binary features enable models to learn differential pricing for specific categories (e.g., imported cars command premium prices).

\subsubsection{Ordinal Features (3)}

\begin{lstlisting}[language=Python, caption=Ordinal Features]
luxury_brands  = ['Mercedes', 'Bmw', 'Audi', 'Lexus', 'Porsche']
premium_brands = ['Toyota', 'Honda', 'Hyundai', 'Kia', 'Nissan']

def brand_tier(make):
    if any(lux  in make for lux  in luxury_brands):  return 3
    elif any(prem in make for prem in premium_brands): return 2
    else: return 1

df['brand_tier'] = df['make'].apply(brand_tier)
\end{lstlisting}

Similarly: \texttt{city\_tier} (1=Metro, 2=Major, 3=Other) and \texttt{engine\_category} (Small/Medium/Large/V.Large).

\subsubsection{Interaction Features (2)}

\begin{lstlisting}[language=Python, caption=Feature Interactions]
df['age_mileage_interact']  = df['car_age'] * df['mileage_km']
df['brand_engine_interact'] = df['brand_tier'] * df['engine_cc']
\end{lstlisting}

\textbf{Rationale:} High mileage on an old car depreciates value more than high mileage on a new car. Luxury brands with larger engines command disproportionate premiums.
